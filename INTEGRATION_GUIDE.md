## 🔧 集成方案

由于调度器运行在 **Node.js 环境**，而 EasyClaw 的 `browser` 工具只能在 **agent 上下文**中使用，需要选择以下集成方式：

### 方案1：通过 Sub-agent 执行 ⭐ 推荐

**架构：**
```
定时任务 (Cron)
    ↓
调度器 (ctrip-scheduler.js)
    ↓
识别需要检查的任务
    ↓
通知 "携程通" Sub-agent
    ↓
携程通执行浏览器检查
    ↓
更新价格并发送通知
```

**实现方式：**

1. 调度器识别需要检查的任务
2. 写入临时文件 `pending-checks.json`
3. "携程通" 定期读取待检查任务
4. 执行浏览器自动化
5. 更新状态并通知用户

**优点：**
- ✅ 充分利用 EasyClaw 能力
- ✅ 可以使用 browser 工具
- ✅ 自动发送飞书通知

**缺点：**
- ⚠️ 需要 "携程通" 持续运行
- ⚠️ 通信通过文件（不够优雅）

---

### 方案2：主 Agent 定时检查 ⭐⭐ 最简单

**架构：**
```
EasyClaw 定时任务 (内置 Cron)
    ↓
主 Agent 读取配置
    ↓
批次执行价格检查
    ↓
更新状态并通知
```

**实现方式：**

创建一个 EasyClaw 定时任务（类似 HEARTBEAT），让主 agent 定期执行价格检查。

**文件：`PRICEMONITOR.md`**
```markdown
# 携程价格监控定时任务

读取 ctrip-monitor-state.json
筛选需要检查的任务
批次执行浏览器检查
更新价格并发送通知
```

**优点：**
- ✅ 最简单，纯 EasyClaw 方案
- ✅ 不需要外部调度器
- ✅ 完全集成

**缺点：**
- ⚠️ 依赖 EasyClaw 运行
- ⚠️ 智能调度功能需要在 agent 中实现

---

### 方案3：Puppeteer 直接集成

**架构：**
```
定时任务 (Cron)
    ↓
调度器 (Node.js + Puppeteer)
    ↓
直接控制浏览器
    ↓
检查价格并更新
    ↓
调用飞书 API 发送通知
```

**实现方式：**

在调度器中集成 Puppeteer，直接控制浏览器。

**优点：**
- ✅ 完全独立运行
- ✅ 不依赖 EasyClaw
- ✅ 灵活性高

**缺点：**
- ⚠️ 需要安装 Puppeteer
- ⚠️ 无法复用 EasyClaw 的浏览器状态
- ⚠️ 需要重新实现登录逻辑

## 🎯 推荐实施方案

### 阶段1：快速原型（现在）

**使用方案2：主 Agent 定时检查**

创建 `PRICEMONITOR.md`，让主 agent 定期执行：

```markdown
# 携程价格监控

## 执行逻辑

1. 读取 workspace/ctrip-monitor-state.json
2. 筛选需要检查的任务（根据 lastCheckedAt + frequencyMinutes）
3. 检查是否在夜间暂停时段（02:00-06:00）
4. 按优先级排序（入住日期近的优先）
5. 分批执行（每批15个）
6. 每个任务：
   - 打开携程页面
   - 提取价格
   - 对比阈值
   - 如果低于目标价，发送飞书通知
   - 更新 lastCheckedAt 和 lastPrice
7. 保存状态

## 检查条件

- 当前时间 >= lastCheckedAt + frequencyMinutes
- 不在夜间暂停时段
- 未超过 autoStopDate
```

然后设置 EasyClaw 定时任务每8小时触发一次。
