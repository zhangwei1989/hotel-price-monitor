## 一、项目定位

PriceWatcher Dashboard 是酒店价格监控系统的**可视化管理界面**，为管理员提供：

- 一览式查看所有监控任务的状态和价格
- 添加、编辑、删除监控任务
- 批量操作任务（暂停/恢复）
- 查看单任务的价格历史趋势图
- 手动触发立即检查

**目标用户：** 张伟（个人使用，单 admin 账户）  
**设计风格：** Vercel 深色主题（`#0a0a0a` 背景，`#111` 卡片，白色强调）

---

## 二、页面结构

```
/login          登录页
/               → 跳转 /tasks
/tasks          任务列表页（主页面）
/tasks/:id      任务详情页
```

### 3.4 添加任务 Modal（新增）

**状态：❌ 未完成（P1）**

触发入口：任务列表页右上角「+ 添加任务」按钮

表单字段：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 酒店名称 | 文本 | ✅ | 非空 |
| 城市 | 文本 | ✅ | 非空 |
| 携程链接 | 文本 | ✅ | URL 格式，含 hotels.ctrip.com |
| 入住日期 | 日期 | ✅ | > 今天 |
| 退房日期 | 日期 | ✅ | > 入住日期 |
| 房型名称 | 文本 | ✅ | 非空 |
| 价格方案提示 | 文本 | - | 如"不含早" |
| 目标价（阈值） | 数字 | ✅ | > 0 |
| 检查频率 | 数字/预设 | ✅ | >= 10 分钟 |
| 自动停止日期 | 日期 | - | >= 退房日期 |

提交后：
1. 调用 `POST /api/tasks`
2. 成功则关闭 Modal 并刷新列表
3. 失败则展示错误信息

## 六、接口依赖

所有接口均通过 `src/api/http.ts`（baseURL `http://127.0.0.1:3001`）调用 PriceWatcher API。

| 前端模块 | 调用的 API |
|---------|-----------|
| 登录 | `/api/auth/login`, `/api/auth/verify` |
| 任务列表 | `/api/tasks`, `/api/task-actions/stats/summary` |
| 任务操作 | `/api/task-actions/:id/*`, `/api/task-actions/batch/*` |
| 任务详情 | `/api/tasks/:id`, `/api/prices/history/:id` |
| 添加任务 | `POST /api/tasks` |
