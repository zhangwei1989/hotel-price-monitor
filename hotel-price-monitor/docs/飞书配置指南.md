# 飞书配置指南

## 第一步:创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 点击"创建企业自建应用"
3. 填写应用名称:如"酒店价格监控"
4. 上传应用图标(可选)

## 第二步:配置应用权限

在应用管理后台,添加以下权限:

### 必需权限
- `im:message` - 发送消息
- `im:message:send_as_bot` - 以机器人身份发消息
- `bitable:app` - 访问多维表格

### 可选权限
- `contact:contact.base:readonly` - 读取通讯录(用于获取用户信息)

## 第三步:获取凭证

在"凭证与基础信息"页面,复制:
- **App ID**: `cli_xxx`
- **App Secret**: `xxx`

## 第四步:创建多维表格

### 方法 1: 手动创建

1. 在飞书中创建一个多维表格
2. 添加以下字段:

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 任务ID | 自动编号 | 自动生成 |
| 酒店名称 | 文本 | 例: 上海外滩W酒店 |
| 酒店ID | 文本 | 携程酒店ID |
| 房型名称 | 文本 | 例: 豪华大床房 |
| 房型ID | 文本 | 携程房型ID |
| 监控日期 | 日期 | 入住日期 |
| 离店日期 | 日期 | 默认为入住日期+1天 |
| 价格阈值 | 数字 | 触发告警的价格上限 |
| 当前价格 | 数字 | 最新查询到的价格 |
| 最后更新时间 | 日期时间 | 自动更新 |
| 状态 | 单选 | 监控中/已触发/已暂停 |
| 通知对象 | 人员 | 选择要通知的用户 |

3. 获取表格凭证:
   - 打开表格,从 URL 中提取 `app_token` 和 `table_id`
   - URL 格式: `https://xxx.feishu.cn/base/{app_token}?table={table_id}`

### 方法 2: API 创建(推荐)

使用飞书 API 自动创建表格,保证字段一致性。

## 第五步:配置环境变量

复制 `.env.example` 为 `.env`,填入配置:

```bash
cp .env.example .env
```

编辑 `.env`:
```env
CTRIP_API_KEY=你的携程API_KEY
CTRIP_API_SECRET=你的携程API_SECRET

FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=xxx

FEISHU_TABLE_APP_TOKEN=从URL中提取
FEISHU_TABLE_ID=从URL中提取

PORT=3000
```

## 第六步:授权表格访问

1. 在表格设置中,点击"高级设置" → "开放接口"
2. 添加你的应用,授予"读写"权限

## 第七步:配置飞书自动化(可选)

### 自动化流程设置

1. 在多维表格中,点击"自动化"
2. 创建新流程:
   - **触发器**: 定时触发(每小时)
   - **动作**: 发送 HTTP 请求
     - URL: `http://你的服务器:3000/api/monitor/run`
     - 方法: POST
     - Header: `Content-Type: application/json`

这样可以无需服务端定时任务,由飞书自动化触发。

## 第八步:测试配置

运行测试脚本:
```bash
npm run dev
```

访问测试接口:
```bash
# 测试通知
curl -X POST http://localhost:3000/api/test/notify \
  -H "Content-Type: application/json" \
  -d '{"userId": "ou_你的open_id"}'
```

## 常见问题

### 如何获取用户的 open_id?

方法 1: 通过飞书机器人对话
- 让用户发送消息给机器人
- 从 webhook 回调中获取 `open_id`

方法 2: 使用通讯录 API
```javascript
const users = await feishuAPI.client.contact.user.list();
```

### 权限不足怎么办?

访问权限申请链接:
```
https://open.feishu.cn/app/cli_a92b495c63b8dcd9/auth?q=需要的权限
```

例如:
```
https://open.feishu.cn/app/cli_a92b495c63b8dcd9/auth?q=im:message,bitable:app
```

### 如何调试 API?

启用详细日志:
```javascript
const client = new lark.Client({
  appId,
  appSecret,
  loggerLevel: lark.LoggerLevel.debug
});
```
