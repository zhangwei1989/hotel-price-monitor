# 携程监控任务调度器

## 功能设计

### 1. 批次调度
- 将所有任务分成多批（每批 15 个）
- 批次间随机延迟 5-10 分钟
- 避免同时大量请求

### 2. 优先级队列
- **高优先级**：入住日期 ≤ 2 天
- **中优先级**：入住日期 3-7 天
- **低优先级**：入住日期 > 7 天

### 3. 智能频率调整
- 价格稳定 48h → 延长间隔到 12 小时
- 接近入住日 → 缩短间隔到 4 小时
- 夜间暂停（02:00-06:00）

### 4. 失败处理
- 重试 3 次（指数退避：60s, 120s, 240s）
- 失败超过 5 次 → 标记为异常，发送告警
- 自动切换 IP（如果启用代理）

## 实现方式

### 选项A：定时任务 (Cron)
```bash
# 每 8 小时执行一次主调度器
0 */8 * * * cd ~/.easyclaw/workspace && node ctrip-scheduler.js

# 或使用 EasyClaw 内置调度
```

### 选项B：Sub-agent 持续监控
```javascript
// 创建专门的调度 sub-agent
sessions_spawn({
  label: "携程调度器",
  task: "定期执行批次调度，管理所有监控任务"
})
```

### 选项C：云函数（Serverless）
```
部署到阿里云/AWS Lambda
定时触发
无需本地运行
```

## 监控指标

- 成功率：目标 > 90%
- 平均响应时间：< 5秒
- 失败重试次数
- IP 封禁次数
- 每日请求总数
